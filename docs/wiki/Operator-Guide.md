# Operator Guide

This guide is for the operator running a Radar Vital session. It assumes the
hardware is assembled, the trainer is installed, and the legal draft has been
reviewed with the research team.

## First run

1. Read the terms and privacy notice shown by the app.
2. Accept only if the draft wording is approved for your study context.
3. Create an operator profile and save the recovery code if the build shows
   one. Recovery codes are shown once and should be stored offline.
4. Complete or skip the onboarding tutorial after you understand the controls.

## Before each session

- Confirm the subject has given informed consent for measurement.
- Use subject codes instead of full names where possible.
- Place the radar at chest height, facing the torso.
- Keep the subject within the tested placement range, ideally 0.4 m to 1.5 m.
- Remove unnecessary motion near the radar and avoid reflective clutter.
- Confirm the Home preflight shows the trainer, serial device, and signal state.

## Running a session

1. Unlock the operator station.
2. Confirm the dashboard is connected to the intended trainer, not demo mode.
3. Open Home and wait for subject detection.
4. Start the session only after the signal stabilizes.
5. Watch live HR, RR, presence, SQI, and stale/fault indicators during capture.
6. Stop the session cleanly and review the report before export.

Do not hide stale, invalid, or rejected signal states in notes. Those states are
part of the research record and affect whether a session is analysis-ready.

## After the session

Review the Report page, export artifacts as required by the thesis protocol, and
delete test sessions that should not be retained. If you open a GitHub issue,
review diagnostics first; logs should not include subject names or private notes.
