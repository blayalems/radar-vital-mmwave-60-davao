# Operator console localization (i18n)

This document describes the lightweight runtime localization layer used by the Angular operator console.

## Files

| File | Responsibility |
| --- | --- |
| `web/src/app/services/i18n.messages.en.ts` | English source catalog and catalog types. |
| `web/src/app/services/i18n.service.ts` | Locale registry, fallback, interpolation, and plural selection. |
| `web/src/app/i18n/translate.pipe.ts` | Standalone template pipe. |

## Principles

- English is the source catalog.
- Additional locales may be partial; missing entries fall back to English.
- Unknown keys fall back to the key string so the UI never renders blank.
- Interpolation placeholders use `{name}`.
- Plural selection uses `Intl.PluralRules` for the active locale and falls back to `.other` when a locale-specific category is unavailable.

## Externalizing a string

Add a key to `EN_MESSAGES`:

```ts
"home.systemChecks": "System checks",
```

Use the standalone pipe in a component template:

```html
<h2>{{ 'home.systemChecks' | translate }}</h2>
```

The `{{ ... }}` delimiters are required for text-node interpolation. Attribute bindings already provide an Angular expression context:

```html
<button
  [attr.aria-label]="'home.rerunCheck' | translate:{ label: check.label }">
</button>
```

In component code:

```ts
private readonly i18n = inject(I18nService)

const message = this.i18n.translate("state.connected")
```

## Pluralization

Define category keys. At minimum, provide `.one` and `.other`:

```ts
"home.checksNeedReview.one": "{count} check needs review",
"home.checksNeedReview.other": "{count} checks need review",
```

Resolve them with:

```ts
this.i18n.plural("home.checksNeedReview", failing.length)
```

`plural()` always uses the numeric argument as `{count}`; caller parameters cannot override it.

## Adding a locale

```ts
const i18n = inject(I18nService)

i18n.registerCatalog("fil", {
  "nav.home": "Tahanan",
  "nav.settings": "Mga Setting",
})

i18n.setLocale("fil")
```

Catalog keys are type-checked against `MessageKey`, which catches spelling mistakes during development.

## Rollout order

1. Primary navigation
2. Top bar
3. Dashboard headings and actions
4. State and status messages
5. Shared dialogs
6. Consent and legal copy, only after preserving the exact legal meaning

## Translation-safe layout

- Avoid widths derived from English text length.
- Let labels wrap with `.r-label` where appropriate.
- Verify localized strings across the documented responsive breakpoints and safe-area insets.
- Bottom-navigation labels may wrap to two lines; do not reintroduce ellipsis clipping.
