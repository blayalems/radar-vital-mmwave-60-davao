# Operator console localization (i18n)

This document describes the lightweight runtime localization layer used by the Angular operator console.

## Files

| File | Responsibility |
| --- | --- |
| `web/src/app/services/i18n.messages.en.ts` | English source catalog and catalog types. |
| `web/src/app/services/i18n.service.ts` | Locale registry, fallback, interpolation, and plural selection. |
| `web/src/app/i18n/translate.pipe.ts` | Standalone pure template pipe. |

## Principles

- English is the source catalog.
- Partial locales fall back to English per key.
- Unknown keys fall back to the key string.
- Interpolation placeholders use `{name}`.
- Plural selection uses `Intl.PluralRules` when available and deterministic English one/other categories otherwise.
- Seed catalog values mirror the current English UI copy.

## Externalizing a string

Add a key:

```ts
"home.systemChecks": "Hardware Preflight Pipeline",
```

Import the standalone pipe and expose the service:

```ts
imports: [TranslatePipe]
protected readonly i18n = inject(I18nService)
```

Pass the locale signal value as the pure-pipe cache key:

```html
<h2>{{ 'home.systemChecks' | translate:i18n.locale() }}</h2>

<button
  [attr.aria-label]="'home.rerunCheck' | translate:i18n.locale():{ label: check.label }">
</button>
```

This keeps locale changes reactive without evaluating translations on every unrelated change-detection cycle.

Direct service use remains available:

```ts
const message = this.i18n.translate("state.connected")
```

## Pluralization

```ts
"home.checksNeedReview.one": "{count} check needs review",
"home.checksNeedReview.other": "{count} checks need review",

this.i18n.plural("home.checksNeedReview", failing.length)
```

The numeric argument always supplies `{count}`; caller parameters cannot override it.

## Adding a locale

```ts
i18n.registerCatalog("fil", {
  "nav.home": "Tahanan",
  "nav.settings": "Mga Setting",
})
i18n.setLocale("fil")
```

Catalog keys are type-checked against `MessageKey`.

## Rollout

The current change establishes and tests the infrastructure. Broad template externalization remains incremental follow-up work. When adopting the pipe, preserve the current English copy so the change is behavior-neutral.

## Translation-safe layout

- Avoid widths derived from English text length.
- Let labels wrap where appropriate.
- Verify localized strings across supported breakpoints and safe-area insets.
- Bottom-navigation labels may use a second line, then deliberately truncate exceptionally long copy to preserve the fixed-height bar.

## Validation

```bash
npm run test:unit:web
npm run build:web
npm run build:check
npm test
```

The unit command runs source-integrity checks, resolves real theme tokens for contrast validation, and executes Angular/Vitest coverage. The contrast matrix checks both `--rv-muted` and `--rv-dim` across neutral and colored container surfaces for all palettes, Night, and High Contrast.

Commit the regenerated `radar_vital_live_dashboard_v12_for_v16_0.html` whenever `web/` changes. `npm run build:check` verifies synchronization.
